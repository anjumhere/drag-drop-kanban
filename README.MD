## ğŸ§  How It Works

### Safe DOM Access with `DOMContentLoaded`

All JavaScript runs inside a `DOMContentLoaded` listener. This guarantees the HTML is fully parsed before any element is queried â€” preventing `Cannot read properties of null` errors that occur when scripts run before the DOM is ready.

```js
document.addEventListener("DOMContentLoaded", () => {
  // All code lives here â€” the DOM is guaranteed to exist
});
```

### Task Creation

Clicking **+ New Task** opens a modal where you enter a title and description. On submit, the task is dynamically built with `createElement` and appended to the **TO DO** column. Each task gets Edit and Delete buttons wired up at creation time.

```js
function createTaskElement(title, description, id) {
  const task = document.createElement("div");
  task.classList.add("task");
  task.draggable = true;
  task.dataset.id = id;
  // builds heading, paragraph, action buttons
  // returns completed element
}
```

### Task Editing âœï¸

The same modal used for creating tasks handles editing too. Clicking **Edit Task** on any card pre-fills the modal with that task's current content. An `editMode` flag on the submit button tells it whether to create a new task or save changes to an existing one.

```js
// The submit button checks a flag to decide its behavior
addTaskBtn.addEventListener("click", () => {
  if (addTaskBtn.dataset.editMode === "true") {
    saveEditedTask(); // update existing card in place
  } else {
    addNewTask(); // create a new card
  }
});
```

> **Why not recreate the task element on edit?**  
> Updating `.textContent` directly on the existing element is more efficient and avoids re-attaching event listeners. The card stays in its column, at its position â€” only the text changes.

### Drag & Drop

Each column listens for five drag events: `dragstart`, `dragenter`, `dragleave`, `dragover`, and `drop`.

- **`dragstart`** â€” stores the dragged task element in `dragItem`
- **`dragenter`** â€” adds a `hover-over` glow class to the target column
- **`dragleave`** â€” removes the glow **only** when truly leaving the column, not when crossing child elements, using `element.contains(e.relatedTarget)`
- **`dragover`** â€” calls `e.preventDefault()` to allow the drop event to fire
- **`drop`** â€” appends `dragItem` into the target column's `.column-body`

```js
// The key fix for child element flickering during drag
column.addEventListener("dragleave", (e) => {
  if (!column.contains(e.relatedTarget)) {
    column.classList.remove("hover-over");
  }
});
```

Event delegation is used for `dragstart` and `dragend` on the `document` â€” since tasks are created dynamically, this avoids needing to attach listeners to elements that don't exist yet.

### Search with Debouncing

The search uses **debouncing** â€” it waits 400ms after the user stops typing before scanning the DOM. This avoids unnecessary queries on every keystroke. Matching tasks get a `highlight` class that adds a glowing teal border, which removes itself after 2.5 seconds.

```js
searchTimeout = setTimeout(() => {
  document.querySelectorAll(".task").forEach((task) => {
    const title = task.querySelector(".title")?.textContent.toLowerCase() || "";
    const desc =
      task.querySelector(".paragraph")?.textContent.toLowerCase() || "";

    if (title.includes(searchValue) || desc.includes(searchValue)) {
      task.classList.add("highlight");
      setTimeout(() => task.classList.remove("highlight"), 2500);
    }
  });
}, 400);
```

### Live Task Counter

`updateTaskCounters()` runs after every add, delete, drag, or edit event. It uses `querySelectorAll(".task")` on each column, which works on dynamically created elements since they're part of the DOM once appended.

```js
function updateTaskCounters() {
  todoColumn.querySelector(".count").textContent =
    todoColumn.querySelectorAll(".task").length;
  progressColumn.querySelector(".count").textContent =
    progressColumn.querySelectorAll(".task").length;
  doneColumn.querySelector(".count").textContent =
```

### CSS Transitions

Transitions are placed on the **base element**, not on the hover/active class. This is critical â€” if the transition is only on the modifier class, it gets removed along with the class and the exit animation snaps instantly.

```css
.task {
  transition:
    0.2s ease-in-out,
    border-color 0.5s ease-in-out,
    box-shadow 0.5s ease-in-out;
}
```
